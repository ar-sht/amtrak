Amtrak Network Hub Vulnerability Analysis

A network-theory analysis of structural vulnerabilities in the U.S. passenger rail system

1. Data Sources

The analysis is built on four data files that describe the Amtrak passenger rail network:

FileFormatContents
station-data.xlsxExcelStation codes with longitude/latitude coordinates
route-data.xlsxExcelNames of all Amtrak routes
route-stop-data.txtPlain textOrdered list of station stops for each route
ridership-data.txtPlain textAnnual ridership figures for each route

How the data is structured

station-data.xlsx provides a row per station with three fields: a short alphanumeric Code (e.g. CHI for Chicago Union Station, NYP for New York Penn Station), and its geographic coordinates (lon, lat).

route-stop-data.txt lists each route by name followed by its stations in order. Each station line has the format City, State - Station Name (CODE). The three-letter code in parentheses links each stop to the station coordinate table.

ridership-data.txt provides a row per route with multiple ridership metrics. The fifth column (total annual ridership in thousands) is used as the route weight.

Network construction

The Amtrak system is modeled as a weighted undirected graph where:

  • Nodes represent stations (537 total).
  • Edges connect consecutive station pairs along each route. If Station A and Station B are adjacent stops on any route, an edge exists between them.
  • Edge weights represent ridership intensity. For a given route, the total route ridership is divided equally among its segments (total ridership ÷ number of stops). When multiple routes share the same segment (e.g. the Acela and Northeast Regional both run NYP→NWK), their weights are summed. This produces a single combined ridership weight per edge.

The resulting graph contains 537 stations and 612 edges, with a largest connected component of 523 stations (97.4% of the network). The remaining 14 stations form two small isolated components that reflect real gaps in the rail network:

  • Downeaster component (12 stations): Boston North Station (BON) through Brunswick, ME. Disconnected from the main network because Boston North and Boston South are separate terminals on opposite sides of the city with no direct through-track.
  • Auto Train component (2 stations): Lorton, VA (LOR) and Sanford, FL (SFA). The Auto Train is a point-to-point service with no intermediate stops, so it appears as an isolated pair.

These disconnected stations are excluded from the largest-component robustness analysis (denominator = 523).

2. Network Theory Metrics: Definitions

2.1 Betweenness Centrality

Definition: Betweenness centrality measures how often a node lies on the shortest path between every other pair of nodes in the network. Formally, for a node $v$:

$$C_B(v) = \sum_{s \neq v \neq t} \frac{\sigma_{st}(v)}{\sigma_{st}}$$

where $\sigma_{st}$ is the total number of shortest paths from node $s$ to node $t$, and $\sigma_{st}(v)$ is the number of those paths that pass through $v$. The value is normalized to the range $[0, 1]$.

Intuition: A station with high betweenness centrality is one that acts as a critical bridge — if you need to travel between many pairs of cities, your shortest route will pass through this station. Removing it forces passengers onto longer detours or severs connectivity entirely.

Why we chose it: Betweenness centrality directly captures a station's role as an interchange or bottleneck. Unlike degree (which only counts direct connections), betweenness reveals stations that control flow through the network — exactly what matters for vulnerability analysis.

2.2 Degree

Definition: The degree of a node is the number of edges connected to it — i.e., the number of other stations it has a direct rail link to.

Intuition: A degree-1 station is a terminus (end of the line). A degree-2 station is a simple pass-through. Stations with degree 3+ are junctions where multiple routes converge.

2.3 Largest Connected Component

Definition: A connected component is a maximal subgroup of stations where every station can reach every other station via some path. The largest connected component is the biggest such group.

Intuition: If you remove a critical station and the network fractures, the largest connected component tells you how many stations can still communicate with each other. A drop in this number means parts of the network have been isolated.

2.4 Edge Weight (Ridership)

Definition: The combined annual ridership (in thousands of passengers) flowing through a given track segment, summed across all routes that share that segment.

Intuition: High-weight edges represent the busiest corridors. Disruption to high-weight edges has the greatest direct impact on passenger volume.

3. How We Computed Each Metric

Betweenness centrality

Computed using NetworkX's betweenness_centrality() function over the full graph with edge weights that reflect passenger flow. Because our raw edge weights encode ridership (higher = busier corridor) but NetworkX treats the weight attribute as a Dijkstra cost (higher = longer path), we invert the raw weight into a cost: cost = 1 / ridership. Busy corridors then become “short” and shortest paths preferentially route through high-ridership segments. Centrality scores are normalized to $[0, 1]$ and every one of the 537 stations receives a score; we rank them and extract the top 10.

Note on methodology: we initially passed raw ridership as the weight, which NetworkX interprets as distance (higher = further apart) — the opposite of what we intended. Inverting ridership to a cost gives the semantically correct flow-weighted centrality. The unweighted sensitivity check in Section 4 confirms the top 8 stations are robust to this choice.

Targeted removal experiment

Starting from the intact network, we sequentially remove the station with the highest betweenness centrality, then the second-highest, and so on up to 10 removals. After each removal, we measure the size of the largest connected component. This simulates a worst-case coordinated attack on the network's most critical nodes.

Random removal experiment

For comparison, we repeat the removal process but select stations uniformly at random. Because random outcomes vary, we average the results over 100 independent trials. After each removal step (1 through 10), we record the mean largest-component size across all trials.

Degree distribution

We count the degree of every station in the graph and tally how many stations have each degree value (1, 2, 3, …, 9).

Edge weight distribution

We collect the ridership weight of all 612 edges and plot their distribution on a log scale to visualize the long tail from low-traffic rural segments to high-traffic urban corridors.

Limitations

  • Ridership is split uniformly along each route's segments (route ridership ÷ number of stops). Real passenger flows are endpoint-heavy — most riders board at major stations and ride partway — but station-level origin-destination data is not publicly available. This likely over-weights small rural middle-of-route segments and under-weights urban endpoints.
  • Edge weights represent annual ridership only. Train frequency (e.g. Empire Service running many daily trains vs. Cardinal running three per week) is not modeled as a separate factor, though it is partially captured by ridership totals.
  • Robustness is measured by largest-connected-component size, which treats “still connected” and “connected via a 3× detour” identically. A more refined measure (e.g. network efficiency) would penalize lengthened paths, but component size is the standard intro-networks metric and suffices for the targeted-vs-random comparison.

Notable patterns

  • Chicago (CHI) has the highest centrality at 0.61, roughly 1.4× the next-highest station (CLE at 0.44) and 25× the network median (0.024).
  • The top 10 all lie along a single east–west flow axis from Chicago through Indiana, Ohio, and upstate New York. The Lake Shore Limited and Empire Service together carry so much of the network's passenger flow that nearly every station along that chain ends up with near-identical centrality.
  • Ranks 2–10 are very tightly bunched (0.42–0.44, a spread of less than 0.02). The meaningful distinction is between CHI, this corridor group, and everything else.

Sensitivity check

To verify that the top-10 ranking is not an artifact of our specific weighting choice, we also computed betweenness under an unweighted model (pure topology, every edge has equal cost). The top 8 stations are identical in both schemes: CHI, CLE, SOB, EKH, WTI, TOL, SKY, ELY. Only ranks 9–10 differ (BUF/SDY under flow-weighting vs. NPV/SAC under unweighted). This confirms the main finding — CHI as the dominant hub with a linear corridor of critical stations to its east — is a property of the network's structure, not of the weighting scheme.

Note on network type

The degree distribution is heavy-tailed with one extreme outlier (CHI at degree 9), consistent with a hub-and-chain topology. With only 537 nodes we do not claim a formal power-law fit; the distribution is better described qualitatively as “one dominant hub, a handful of minor junctions, and long linear chains” than as a true scale-free network in the Barabási–Albert sense.

5. What Each Graph Shows

Graph 1: Targeted vs Random Station Removal

What it shows: Two curves plotted against the number of stations removed (0–10). The red solid line shows what happens when stations are removed in order of betweenness centrality (targeted attack). The blue dashed line shows what happens when stations are removed randomly (averaged over 100 trials). The shaded pink region between them highlights the “vulnerability gap.” The right y-axis shows the percentage of the original network remaining.

Key observations:

  • Removing just CHI drops connectivity to 95% (499 stations). The next several removals (CLE through ELY) hold the component steady at ~94% (491 stations) — these stations are all adjacent along the same corridor, so removing one already cuts off the segment the next would have influenced.
  • After 10 targeted removals, 459 stations (87.8%) remain connected.
  • Random removal of 10 stations preserves ~92% connectivity.
  • The “vulnerability gap” between targeted and random removal is therefore only about 4 percentage points. When centrality is correctly weighted by passenger flow, the top-10 stations are not global articulation points — they cluster on one corridor, so removing them just chips away at that corridor rather than fragmenting the entire network. (Removing CHI alone remains the single most damaging action.)

Graph 2: Station Degree Distribution

What it shows: A bar chart showing how many stations have each degree value (1 through 9). Each bar is labeled with the count and percentage.

Key observations:

  • 416 stations (77%) have degree 2 — they are simple pass-through stops on a single line.
  • 17 stations (3%) have degree 1 — route termini.
  • Together, 81% of stations have degree ≤ 2. The network is overwhelmingly composed of linear chains.
  • Only 3 stations have degree ≥ 6, and exactly 1 station (CHI) has degree 9.
  • This extreme degree concentration confirms the network's hub-and-spoke topology.

Graph 3: Betweenness Centrality Distribution

What it shows: A histogram of betweenness centrality across all 537 stations. The dashed vertical red line marks the top-10 threshold. The top 3 outliers (CHI, CLE, SOB) are individually labeled. An inset box reports the median and maximum values.

Key observations:

  • The distribution is extremely right-skewed: the median centrality is 0.024, while the maximum (CHI) is 0.61 — 25× the median.
  • The vast majority of stations cluster near zero centrality. Most stations are not structurally important.
  • There is a clear gap between the “body” of the distribution and the top-10 outliers, confirming that hub importance is not a continuum but a sharp divide between critical and non-critical stations.

Graph 4: Geographic Hub Map

What it shows: A map of the continental United States with all 537 stations plotted as points. The top 10 hubs are shown as large circles sized and colored by centrality (darker red = higher centrality), with labeled leader lines showing rank and station code. All other stations appear as small grey dots. Edges are drawn with thickness proportional to ridership.

Key observations:

  • All 10 top hubs lie along a single east–west corridor from Chicago through Indiana and Ohio to upstate New York (BUF, SDY) — essentially the Lake Shore Limited / Empire Service flow axis. A regional event (severe winter weather, Great Lakes storm, infrastructure failure) along this corridor could simultaneously disable multiple top hubs.
  • Despite the dense station coverage in the Northeast Corridor (visible as a thick line of dots from Boston to Washington), no NEC station appears in the top 10 — the NEC's mesh-like topology means no single station there is a bottleneck on the flow-weighted shortest paths.
  • No Southern or Western station appears in the top 10 under flow-weighted centrality. Those routes carry low passenger volume, so flow-weighted shortest paths rarely traverse them.
  • The Western United States has sparse, long-distance routes with minimal redundancy, but their low ridership keeps their centrality scores low under this weighting.

6. Insights & Implications

Insight 1: Chicago Union Station is the dominant hub

Chicago's betweenness centrality of 0.61 is roughly 1.4× the next-highest station and 25× the network median. It is the only station with degree 9, serving as the sole interchange between nearly all long-distance routes. Removing CHI alone drops the largest connected component by 24 stations in a single step — the single most damaging removal anywhere in the network. Any disruption at Chicago (weather, infrastructure failure, labor action, security incident) materially degrades national connectivity.

Implication: Prioritize infrastructure hardening and redundancy planning for Chicago Union Station. Develop pre-positioned contingency plans (bus bridges, rerouting protocols) specifically for CHI outages. Consider investment in at least one bypass route that connects Eastern and Western networks without passing through Chicago.

Insight 2: Targeted removal of the top hubs is only marginally worse than random

Removing the top 10 stations by flow-weighted betweenness leaves 459 stations (87.8%) connected; removing 10 random stations leaves ~482 (92%). The gap is only about 4 percentage points. The reason: the top 10 all lie along a single east–west corridor, so successive removals are redundant — each one cuts off the segment the next would have influenced, rather than severing a new part of the network. The glaring exception is CHI itself, which is a genuine articulation point; its removal alone accounts for most of the targeted loss.

Implication: Resilience investment should be disproportionately concentrated on CHI, not spread evenly across the top 10. Beyond CHI, the real vulnerability is the corridor (Chicago–Cleveland–Buffalo–Albany), not the individual stations along it — any single point can be bypassed as long as the corridor is intact.

Insight 3: The Chicago–Albany corridor is the structural Achilles’ heel

All 10 top hubs lie along a single east–west flow axis: Chicago → northern Indiana → Ohio → Buffalo → Schenectady. This is the combined Lake Shore Limited and Empire Service path, and it carries enough passenger flow that nearly every station on it is at the top of the flow-weighted centrality ranking. A regional disruption along this corridor (severe winter storm, Great Lakes weather, track failure between major cities) could simultaneously affect multiple top stations.

Implication: Develop at least one alternate East–West route that does not traverse the Chicago–Cleveland–Buffalo corridor. Even a single cross-link (for example, a Cincinnati–Pittsburgh connection or a restored Cardinal-style route) would meaningfully reduce the network's dependence on this single long-distance axis.

Insight 4: Most critical stations are bottlenecks, not junctions

Stations like SOB, EKH, SKY, and ELY rank in the top 10 by centrality despite having only 2 connections each. Their importance comes entirely from topology — they sit on the only path between large sections of the network. This means they are fragile by design.

Implication: Adding even one bypass edge around a bottleneck station would dramatically reduce its centrality and improve network resilience. Each such addition would be a low-cost, high-impact improvement. Specifically: a bypass connecting the two neighbors of any degree-2 bottleneck would provide an alternate path and cut that station's criticality roughly in half.

Insight 5: 81% of stations are simple pass-throughs — the network is linear, not meshed

The degree distribution shows the network is composed of long linear chains tied together at a handful of interchange points. This means each route is essentially a single thread: any break in the thread isolates all stations beyond the break point.

Implication: Prioritize creating cross-links between existing parallel routes. Each new cross-link converts a fragile linear chain into a more resilient loop. Focus on areas where routes run geographically close but have no connecting service.

Insight 6: Revenue risk (NEC) and structural risk (Chicago corridor) are different problems

The highest ridership is concentrated in the Northeast Corridor (NYP↔NWK at 240, 36× median), which has mesh-like redundancy and no stations in the top 10 by centrality — a local outage there hurts revenue but doesn't fragment the network. The highest structural criticality is on the Chicago–Albany long-distance axis, which carries modest ridership per segment but sits on the only east–west long-distance path. These are two different failure modes requiring different responses.

Implication: Maintain separate risk frameworks for NEC financial risk (revenue protection, capacity management on a dense mesh) and long-distance structural risk (connectivity of a single thin corridor). A disruption in the NEC primarily affects revenue; a disruption on the Chicago–Albany corridor primarily fragments service across the entire national network.
